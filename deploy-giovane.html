1 Confirmar versão no GitHub

O EasyPanel builda direto do GitHub. Antes do rebuild, confirmar se o commit abaixo já está na branch main.

⚠️ Antes do rebuild: confirmar que o GitHub está depois de 48f385a e contém o fix ajusta instalacao ios fora do safari. Se ainda estiver em 48f385a, falta push do Eduardo antes do rebuild.
💡 Repositório correto: portaleducare1-glitch/educare-kit-pedagogico. Branch: main. O repo já tem Dockerfile e nginx.conf prontos na raiz.
2 Criar serviço no EasyPanel
  1. Abrir EasyPanel

    Entrar no painel do VPS TurboCloud e ir em Projetos → Novo Projeto (ou dentro de um projeto existente da Educare).

  2. Adicionar serviço → App

    Selecionar App e depois GitHub como fonte.

  3. Conectar o repositório

    Selecionar a conta portaleducare1-glitch e o repositório educare-kit-pedagogico.
    Branch: main.

  4. Configurar o build

    O EasyPanel detecta o Dockerfile automaticamente. Deixar como Dockerfile (não Nixpacks).

    Build type: Dockerfile
Dockerfile path: ./Dockerfile
Build context: . (raiz do projeto)
  5. Porta exposta

    Definir porta como 80 (o nginx escuta na 80).

  6. Sem variáveis de ambiente

    Nenhuma variável de ambiente é necessária. O projeto é 100% estático.

  7. Deploy

    Clicar em Deploy. O EasyPanel vai clonar o repo, rodar o build Docker (~2 min) e subir o container.

3 Configurar domínio e SSL
  1. Adicionar domínio no serviço

    Dentro do serviço criado, ir em Domains → Add Domain e inserir:

    app.educarepedagogia.com.br
  2. Apontar DNS

    No painel de DNS da Educare (provavelmente HostGator ou Cloudflare), criar um registro:

    Tipo:  A
Nome:  app
Valor: [IP do VPS TurboCloud]
TTL:   3600
  3. SSL automático

    O EasyPanel usa Traefik com Let's Encrypt. Assim que o DNS propagar (pode levar alguns minutos), o certificado SSL é emitido automaticamente. Nenhuma configuração adicional necessária.

✅ Domínio e SSL já estão ativos em app.educarepedagogia.com.br/portal. Nesta rodada, basta rebuildar o serviço depois do push correto.
⚠️ HSTS ativo: O nginx já envia o header Strict-Transport-Security. Certifique que o SSL esteja funcionando antes do primeiro acesso para evitar que o navegador bloqueie HTTP.
4 Liberar CORS no WordPress (fazer por último)

O portal abre PDFs hospedados no WordPress da Educare. O domínio definitivo já é https://app.educarepedagogia.com.br. Se mexer no snippet de CORS, usar comparação estrita com esse domínio; não deixar refletindo qualquer origem.

  1. Acessar o WordPress da Educare

    Ir em Code Snippets → Snippet ID 8 (CORS dos materiais pedagógicos).

  2. Usar allowlist estrita do domínio 
    // Produção esperada:
$origin = $_SERVER['HTTP_ORIGIN'] ?? '';
if ($origin === 'https://app.educarepedagogia.com.br') {
    header('Access-Control-Allow-Origin: https://app.educarepedagogia.com.br');
}
  3. Salvar e testar

    Abrir qualquer material no portal e confirmar que o PDF carrega sem erro no console do navegador.

🔒 Fazer esse passo por último, após o domínio estar estável. Trocar antes pode quebrar os testes intermediários.
5 Como fazer atualizações no futuro

Qualquer alteração no código (novos materiais, ajustes de interface, etc.) segue esse fluxo:

  1. Eduardo faz push para a branch main

    Nesta instalação, o webhook/auto-deploy não tem sido confiável. Depois do push, acionar rebuild manual no EasyPanel.

  2. Ou acionar manualmente

    No painel do serviço, clicar em Deploy para forçar um redeploy com a versão mais recente do GitHub.

⚡ O build leva aproximadamente 2 a 3 minutos. O portal fica indisponível por alguns segundos durante a troca de container.

Checklist pos-deploy
  • Portal abre em https://app.educarepedagogia.com.br/portal
  • Cadeado verde (SSL valido) no navegador
  • Navegar para a pagina de acervo e clicar num material
  • PDF abre normalmente (sem erro de CORS no console)
  • Testar no iPhone instalado: sem aviso recorrente de atualização de ícone e sem splash HTML duplicada
  • Testar iPhone fora do Safari: aviso de instalação deve orientar copiar link e abrir no Safari
  • Testar no Android instalado: não deve aparecer mensagem sobre Safari/atualização de ícone
  • Testar botão WhatsApp no fim da página: não deve tampar links do rodapé
  • Recarregar a página numa rota interna (ex: /portal/acervo) — não deve dar 404
  • CORS do WordPress restrito ao dominio definitivo, se o snippet for alterado
  • Favoritar um material e reabrir o portal — favorito persiste
